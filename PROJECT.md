# PROJECT

## Назначение

`AutoGraph` — GUI-приложение на Python для переноса измерительных данных из файлов `TXT` и `S2P` в Excel-книги и Excel-шаблоны.

Основные сценарии:

- импорт одного `TXT`-файла в Excel;
- пакетная обработка папки `TXT`-файлов;
- импорт одного `S2P`-файла в Excel-шаблон;
- пакетная обработка папки `S2P`-файлов;
- обновление формул и графиков внутри заранее подготовленного Excel-шаблона.

## Архитектура

### `autograph_service/gui.py`

Главный пользовательский интерфейс на `Tkinter`. Отвечает за:

- выбор режима `TXT` / `S2P`;
- выбор источника `Один файл` / `Папка`;
- ввод путей к данным и Excel-шаблонам;
- показ или скрытие полей в зависимости от режима;
- валидацию перед запуском;
- фоновое выполнение обработки в отдельном потоке;
- вывод лога и показ итоговых сообщений.

### `autograph_service/service.py`

Внутренний runtime-слой. Отвечает за:

- описание конфигурации запуска через `JobConfig`;
- валидацию конфигурации;
- выбор сценария обработки;
- выполнение одиночных и пакетных `TXT`/`S2P` сценариев;
- передачу логов через callback вместо `print`/`input` как пользовательского интерфейса.

Главный внутренний API:

```python
from autograph_service.service import JobConfig, run_job
```

### `autograph_service/txt_parser.py`

Парсер TXT-файлов измерений:

- читает файл в `cp1251`;
- выделяет метаданные;
- находит заголовок таблицы;
- возвращает строки данных и заголовки.

### `autograph_service/s2p_parser.py`

Парсер `S2P`-файлов формата Touchstone:

- читает файл;
- разбирает строку опций `#`;
- конвертирует частоту в `GHz`;
- рассчитывает `S11/S21/S12/S22` в dB;
- возвращает готовые строки для записи в Excel.

### `autograph_service/excel_handler.py`

Обработчик Excel для TXT-сценария:

- создаёт или открывает книгу;
- копирует шаблонный лист;
- пишет данные и метаданные;
- обновляет лист с графиками;
- при необходимости создаёт дополнительные графики.

### `autograph_service/s2p_excel_handler.py`

Обработчик Excel для S2P-сценария:

- копирует листы и графики из шаблона;
- записывает S-параметры;
- обновляет диапазоны данных графиков;
- автоматически масштабирует оси.

### `autograph_service/main.py`

GUI-first entrypoint внутри пакета. Запускает `Tkinter`-окно.

### `main.py`

Корневой wrapper для запуска GUI из корня репозитория.

## Режимы работы

### Обычный TXT-режим

Без шаблона данные записываются в новый или перезаписываемый лист Excel.

### Шаблонный TXT-режим

С шаблоном проект:

- копирует лист-шаблон;
- пишет данные в табличную часть;
- переносит метаданные в фиксированные ячейки;
- обновляет лист с графиками.

### Шаблонный S2P-режим

`S2P` поддерживается только в шаблонном режиме. Проект:

- создаёт новую выходную книгу;
- копирует шаблонный лист;
- пишет S-параметры;
- при необходимости копирует лист графиков из шаблона;
- обновляет графики и масштабы осей.

### Пакетная обработка

При обработке папки файлы:

- сканируются и сортируются по имени;
- для `TXT` группируются максимум по 8 на одну выходную книгу;
- для `S2P` обрабатываются по одному файлу на результат;
- сохраняются в отдельные результирующие книги.

## Пользовательский интерфейс

### Базовые элементы окна

- выбор типа данных: `TXT` / `S2P`;
- выбор источника: `Один файл` / `Папка`;
- путь к входным данным;
- путь к Excel-файлу или Excel-шаблону;
- чекбокс `Шаблонный режим`;
- кнопки `Запустить` и `Очистить`;
- встроенный лог выполнения.

### Динамические поля

#### TXT, один файл, обычный режим

- `Имя листа`;
- `Выходной файл`.

#### TXT, шаблонный режим

- `Лист-шаблон`;
- `Лист графиков`;
- `Выходной файл` только для одиночного режима.

#### S2P

- `Лист-шаблон`;
- `Лист графиков в шаблоне`;
- `Использовать графики из шаблона`;
- `Выходной файл` только для одиночного режима.

## Способы запуска

Поддерживаются три варианта, и все они открывают GUI:

```bash
python main.py
python -m autograph_service
autograph
```

## Тестирование

Автотесты находятся в `tests/` и покрывают:

- smoke-тесты GUI;
- валидацию формы;
- одиночный и пакетный `TXT`;
- одиночный и пакетный `S2P`.

Команда запуска:

```bash
python -m unittest discover -s tests -v
```

## Ограничения текущего решения

Проект всё ещё жёстко привязан к структуре Excel-шаблонов:

- имена листов;
- координаты ячеек;
- формулы;
- диапазоны графиков;
- ограничение до 8 серий на листе графиков;
- отсутствие GUI-настроек для `create-chart` в первой версии.

Это сознательная доменная привязка текущей реализации, а не архитектурная ошибка.
